# 玻璃厂情绪整合项目说明文档

## 一、项目概述

玻璃厂情绪整合是一个面向心理陪伴与情绪表达场景的多模态交互产品。项目将“玻璃碎片”和“金缮修复”作为核心隐喻，引导用户完成一次从看见自己、表达感受、提炼意象，到重组作品、命名安放和生成情绪音乐的完整体验。

用户可以通过摄像头镜像、语音倾诉或文字输入表达当下状态。系统在浏览器本地识别面部表情，并通过服务端代理完成语音转写、文本意象提炼、作品命名、结束回应和情绪音乐生成。最终，用户的表达会被转化为玻璃碎片、金缮作品和一段 15 秒纯乐器情绪音乐，形成“看见-表达-重组-安放”的仪式化交互。

项目已形成从输入采集、情绪理解、意象生成、艺术化表达、作品沉淀到音乐反馈的端到端体验闭环，重点体现多模态输入、隐私友好的本地识别、LLM 情绪意象提炼、前端艺术化渲染和情绪音乐生成之间的协同能力。

## 二、核心功能

1. 镜像识别：通过浏览器摄像头获取用户面部状态，使用 MediaPipe Face Landmarker 在本地输出 blendshape 系数，并映射为粗粒度情绪、强度、头部姿态和视线信号。
2. 语音倾诉：支持麦克风录音，前端采集音频后通过 Node 服务端代理调用 OpenAI 兼容语音转写接口，避免 API Key 暴露在浏览器端。
3. 文字降级：当摄像头、麦克风或云端转写不可用时，用户仍可通过文字输入完成主流程。
4. 情绪意象提炼：Node 服务调用 LLM，从用户表达中提取意象词、意象注解、颜色、权重、主情绪、强度和情绪摘要；LLM 不可用时使用本地关键词与规则兜底。
5. 玻璃作品生成：前端根据意象、颜色和权重生成玻璃碎片，并通过工作台与熔炉交互将碎片重组为金缮玻璃作品。
6. 作品命名与回应：系统根据意象和主情绪生成候选作品名，并在用户完成作品后生成一句具体、温柔、不诊断的结束回应。
7. 情绪音乐生成：Python 音乐模块根据 7 类情绪和 0-100 强度生成 15 秒 wav 音乐；LLM 生成失败或超时时自动降级到 21 首本地兜底音乐池。

## 三、技术栈

### 1. 前端

- 原生 HTML / CSS / JavaScript，无前端构建工具。
- `public/factory.html`：主交互流程页面，承载门口、镜像、倾诉、工作台、熔炉、命名等场景。
- `public/emotion-engine.js`：封装 MediaPipe Face Landmarker，本地完成面部关键点与表情系数识别。
- `public/voice-input.js`：封装录音、Web Speech 降级、OpenAI 兼容转写和 DashScope 多模态转写能力。
- `public/music-engine.js`：负责播放后端生成的 wav 音乐，并用 Web Audio 合成少量界面音效。
- 实验页面包括 `svg-face.html`、`rive-face.html`、`transcript-test.html` 等，用于表情驱动、Rive 角色和语音转写调试。

### 2. Node.js 服务端

- `server.js` 使用 Node.js 标准库实现，无 Express 等框架依赖。
- 默认监听 `5173`，负责静态资源服务、本地 session 数据读写、语音转写代理、LLM 调用、意象提炼、作品命名、结束回应和音乐任务调度。
- 运行数据保存到 `data/transcripts/`，生成音频保存到 `data/audio/`。
- 通过环境变量配置 LLM 与 STT 通道，包括 `LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL`、`STT_MODEL` 等。

### 3. Python 音乐服务

- `services/music_service.py` 使用 Python 标准库 `ThreadingHTTPServer` 封装本地 HTTP 子服务，默认监听 `5174`，仅绑定 `127.0.0.1`。
- `ember_music/` 是情绪音乐生成模块，暴露 `generate_music()` 和 `healthcheck()`。
- 音乐生成路径包括 LLM 作曲蓝图、规则补全、音色白名单映射、合成器渲染和 wav 输出。
- 音乐模块硬约束包括：每首 15 秒、纯乐器、无环境音、无 vibrato、无 tanh 失真、音色限定在 11 种白名单内。
- Python 依赖见 `ember_music/requirements.txt`：`openai>=1.30.0`、`numpy>=1.24.0`、`scipy>=1.10.0`。

### 4. 数据与接口

主要本地接口包括：

- `POST /api/sessions`：创建一次本地体验 session。
- `PATCH /api/sessions/:id/transcript`：写入语音转写片段。
- `PATCH /api/sessions/:id/text`：写入文字输入。
- `POST /api/sessions/:id/imagery`：生成或兜底提炼情绪意象。
- `POST /api/sessions/:id/artwork-name`：生成作品候选名。
- `POST /api/sessions/:id/closing`：生成结束回应。
- `POST /api/sessions/:id/music`：触发异步音乐生成。
- `GET /api/sessions/:id/music`：轮询音乐生成状态。

## 五、未来规划

1. 情绪识别增强：在现有 blendshape 规则映射基础上，引入更稳定的轻量分类模型或个体校准机制，降低光照、摄像头角度和表情幅度对判断的影响。
2. 心理安全策略完善：补充更严格的高风险表达识别、危机提示、人工帮助入口和内容安全边界，确保产品仅定位于陪伴与表达支持，不替代专业诊疗。
3. 多端与移动体验优化：强化移动端适配、触控交互、低性能设备降级策略和离线演示能力。
4. 音乐生成质量提升：继续优化情绪到编曲配方的映射，扩展兜底音乐池，评估更低成本或更稳定的音乐生成链路。
5. 数据与隐私治理：在正式版本中引入更清晰的用户授权、数据最小化、可删除机制和部署侧密钥管理。
6. 部署与观测：补充生产部署方案、日志指标、LLM 成本统计、音乐生成成功率和降级比例监控。
7. 场景拓展：探索心理陪伴应用、校园心理活动、线下疗愈展陈、艺术工作坊和企业员工关怀等使用场景。

## 六、开源依赖声明

本项目不是直接 fork、复制或套壳某一个完整开源应用；项目的主流程设计、前端交互、Node 本地服务、语音转写代理、情绪意象接口、玻璃作品生成逻辑、音乐服务封装和产品整合均为本团队完成的原创增量工作。

项目使用的第三方开源库、开放模型服务或外部能力主要包括：

| 依赖/能力 | 使用位置 | 用途 | 声明 |
|---|---|---|---|
| MediaPipe Tasks Vision / Face Landmarker | `public/emotion-engine.js` | 浏览器端面部关键点、表情 blendshape 和本地情绪信号识别 | 作为基础视觉识别能力使用，项目未修改其源码 |
| Rive runtime | `public/rive-face.html` | 角色动画实验页 | 仅用于实验性动画驱动演示 |
| Web Speech API | `public/voice-input.js` | 无 API Key 时的浏览器内置语音识别降级 | 由浏览器提供，正式产品需提示其可能调用浏览器厂商服务 |
| OpenAI 兼容接口 | `server.js`、`public/voice-input.js`、`ember_music/` | 语音转写、文本生成、意象提炼、作品命名、结束回应、音乐蓝图生成 | 作为外部模型服务调用，API Key 通过服务端环境变量配置 |
| DashScope 兼容接口 | `public/voice-input.js` | 可选语音转写与语气标签实验 | 作为可替换语音后端，非主流程硬依赖 |
| NumPy | `ember_music/` | 音频数组计算和合成基础 | Python 音乐生成模块依赖 |
| SciPy | `ember_music/` | wav 文件输出和音频处理 | Python 音乐生成模块依赖 |
| Python OpenAI SDK | `ember_music/` | 调用 OpenAI 兼容 LLM 服务 | Python 音乐生成模块依赖 |
| 浏览器 Web Audio API | `public/music-engine.js` | 前端界面音效合成和音乐播放 | 浏览器原生能力 |

项目中 `ember_music/fallback_wavs/` 包含 21 首本地兜底 wav，用于 LLM 音乐生成失败、超时或未配置 API Key 时保持产品流程完整。该兜底池与音乐生成模块一起作为本项目内容的一部分使用。

## 七、本团队原创增量工作说明

本团队围绕“情绪表达与心理陪伴”主题完成的原创增量包括：

1. 设计并实现“玻璃厂”情绪整合叙事流程，将镜像识别、倾诉、意象、碎片、金缮、命名和音乐串联为完整体验。
2. 编写主流程前端页面与多幕交互，包括场景切换、玻璃碎片生成、作品重组、熔炉仪式、命名选择和音乐播放。
3. 封装浏览器端情绪识别引擎，将 MediaPipe 输出转换为项目可用的情绪、强度、视线和姿态信号。
4. 实现语音输入模块，支持云端转写、浏览器 Web Speech 降级、实时音量反馈、最长录音限制和失败回退。
5. 实现 Node 本地服务，完成 session 数据结构、文件持久化、接口路由、LLM 代理、语音转写代理和音乐任务调度。
6. 设计情绪意象提炼 prompt 与本地兜底规则，使短文本、接口失败或未配置 API Key 时仍可继续体验。
7. 实现情绪音乐生成模块的服务化封装、7 类情绪与 3 档强度映射、15 秒 wav 输出和 21 首兜底音乐池降级机制。
8. 整合隐私与安全原则：摄像头画面仅本地识别、不上传画面；API Key 不写入前端；录音转写失败可退回文字输入。

## 八、运行方式简述

基础环境：

- Node.js 18 或更高版本。
- Python 3.10 或更高版本。
- Chrome 或 Edge 浏览器。

安装 Python 依赖：

```bash
python3 -m venv .venv
.venv/bin/pip install -r ember_music/requirements.txt
```

启动主服务：

```bash
node server.js
```

打开浏览器访问：

```text
http://localhost:5173/
```

如需启用 LLM、语音转写和音乐实时生成，需要配置对应环境变量；未配置时项目会尽量使用本地规则、Web Speech 或兜底音乐池完成降级体验。
