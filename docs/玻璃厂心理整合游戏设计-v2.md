# 玻璃厂心灵旅程 · 修正版流程设计 (v2)

## 一句话理念
你被一面会共情的镜子看见 → 你亲手给情绪命名 → 把它取出来变成一块玻璃 → 打碎、重组、金缮/熔铸 → 最终成为一件带着金色裂痕的工艺品。
**「重建」不是抹掉伤痕,而是带着裂痕长出新的形状。**

## 疗愈机制对照(每一幕在"做什么"）
| 幕 | 用户体验 | 背后的心理机制 |
|---|---|---|
| 1 迎接 | 被温柔接住 | 落地 grounding、自我关怀 |
| 2 镜像 | "我的情绪被看见了" | 被看见/被镜映(共情映射) |
| 3 命名 | "这是委屈,不是愤怒" | 情绪命名 affect labeling + 情绪颗粒度 |
| 4 取出/打碎 | 把心里的东西拿到手上 | 外化 externalization(≠发泄) |
| 5 重组·金缮/熔铸 | 我亲手重建 | 赋义、掌控感、创伤后成长 |
| 6 收束 | 被理解、带走一件东西 | 安抚、闭环、回到现实 |

---

## 1. 迎接:玻璃精灵在门口
- 玻璃精灵递上一句**随机暖心话**(冷暖渐变、柔光粒子)。
- 一句轻问:"今天,想让我陪你待一会儿吗?" → 一个按钮进入。
- 目的:降速、建立安全感,把用户从负面情绪里轻轻托一下。
- ⚠️ 底部常驻一行小字安全网:"如果此刻你感到难以承受,可以联系 [危机热线]。" —— 声称疗愈的产品,这行必须有。

## 2. 镜像:被看见(← 直接用 sketch-face.html / emotion-engine.js)
- 摄像头开启,**简笔画角色实时镜映**用户的表情(现成 demo)。
- **关键改动 —— 把"判定"改成"猜测+邀请确认":**
  - 引擎给出的 `emotion` 不再当结论显示,而是变成一句**试探性的、可被否定的**话:
    - 例:检测到 sad → "我好像在你脸上看到一点…沉沉的东西,是这样吗?"
    - 例:检测到 neutral → "此刻你看起来很安静。心里也是吗?"
  - 画面上那张简笔画脸就是"你情绪的形态"——用户看着它跟着自己变,产生"它在陪我 / 它懂我"的感觉。
- 文案原则:**永远给用户否定权**。旁边一直有"不太对,我来说"的入口 → 直接进第 3 幕。
- 技术对接:`onFrame(d)` 里已有 `d.emotion` 和 `d.scores`,把 `EMOTION_META[d.emotion].msg` 换成上面这种试探句即可;`d.scores` 的置信度低时,措辞更谦逊("我不太确定…")。
- 隐私:沿用现有"画面不会上传"的提示,并给一个不开摄像头也能玩的入口(直接跳命名)。

## 3. 命名:从"被看见"到"我来说清楚"
- 承接镜像给的粗粒度情绪,让用户**细化命名**——这是全流程疗效最高的一步。
- 交互:一个"情绪光谱"面板。
  - 第一层沿用引擎的 5 类(开心/难过/惊讶/生气/平静),预选镜像刚才的猜测。
  - 第二层展开**更细的词**(情绪颗粒度是疗效关键):
    - 难过 → 委屈 / 失落 / 孤独 / 疲惫 / 想念
    - 生气 → 不甘 / 被冒犯 / 无力 / 烦躁
  - 允许用户**自己打一个词**。
- 一句**可选的表达性书写**(Pennebaker,别强制):"这份[委屈],是关于______的。"
- 输出:`{ 情绪名, 强度, 一句话 }` → 这组数据**决定下一块玻璃的颜色和质感**(把命名和玻璃因果绑定,隐喻才连贯)。

## 4. 取出 / 打碎:把它从心里拿出来
- **重构框架(最重要的一处文案修正):不是"砸碎发泄",是"取出、看清、承认"。**
  - 承接:"我们把这份[委屈]从你心里,轻轻取出来。" → 生成一块颜色/裂纹对应第3幕的玻璃。
  - 用户点击/长按 → 玻璃**裂开成碎片**(裂纹扩散动画)。
  - 台词:"看,它一直压在心里的样子。现在它在你手上了。"
- ⚠️ 科学依据:纯宣泄(砸东西发泄)会**强化**而非释放负面唤起(Bushman 2002)。所以入口框架必须是"外化+承认",而不是"爽一下"。
- **碎片一定要保留**——它们不是垃圾,是下一步重建的材料。这块设定是"打碎"能变得有建设性的关键桥梁。屏幕上收集起这些碎片:"这些,是我们接下来要用的。"

## 5. 重组 · 金缮/熔铸:我亲手重建(= 你们要的"加工成工艺品")
> 解决旧流程的隐喻断裂:旧版"拼好形状→又扔进火里熔掉"会让人觉得"刚重建又被毁"。
> 修正:**火不是销毁,是把碎片"焊"在一起,并让裂缝显出金色。** = 火中金缮。

- 步骤:
  1. **重组**:把碎片拖进画布,自由拼 or 按提示拼(心 / 花 / 翼 / 星 / 灯…)。每放一块,给一句**具体的、和用户命名挂钩**的正向反馈:"这块'委屈'现在成了翅膀的一角。"
  2. **入炉/金缮**:点"送入熔炉" → 火焰**流动、把碎片熔接**,冷却时**裂缝里流出金线**(kintsugi)。台词:"裂缝没有被藏起来——它们被镀成了金色。"
  3. **成形**:生成一件**带金色裂痕的工艺品**(玻璃器物/挂件/光的形状)。
- 心理意义:掌控感 + 赋义 + "伤痕是我的一部分,而且让我更完整"。这一步是整条弧线的高潮。
- 可行性:火焰/熔化**别做真实物理**,用预渲染动画或 shader/粒子;金线用描边动画即可。

## 6. 收束:被理解,并带走一件东西
- **音乐**:根据第3幕的情绪名切一段**预制 ambient**(黑客松别做实时生成),从"沉"渐渐走到"稳"。
- **一句具体的回应**(不是通用鸡汤):引用用户第3幕写的那句话,给一句针对性的话。
  - 例:用户写"关于总是被忽略" → "被忽略的感觉很真实。今天你没有忽略它,你把它做成了光。"
- **带走**:工艺品可保存/下载,附一行"今日心情"落款。
- **回到现实 + 一点点行动**:"接下来一小时,想为自己做一件小事吗?"(可选,给用户主动权)。
- 再次露出安全网那行。

## 数据流(把六幕串成一条线)
```
镜像 emotion(粗) ──▶ 命名 {情绪名,强度,一句话}
                          │
                          ├─▶ 决定玻璃颜色/裂纹（第4幕）
                          ├─▶ 决定金缮工艺品色调（第5幕）
                          ├─▶ 选择收束音乐（第6幕）
                          └─▶ 生成结尾那句"具体回应"（第6幕）
```
一条数据贯穿始终,用户才会觉得"整个过程都在回应我这一个情绪"。

## 12 小时优先级(先保主线,炫技可降级)
- **P0 必须稳(疗效主线)**:命名交互(第3幕)、外化文案(第4幕)、金缮工艺品(第5幕)、结尾那句具体回应 + 保存(第6幕)。这几样决定"眼眶一热"。
- **P1 已有,接进来**:sketch-face 镜像(第2幕)——改成"猜测+确认"话术,约 1–2 小时。
- **P2 能省则省**:
  - 音乐 → 预制曲库按情绪切,别实时生成。
  - 火焰/熔化 → 预渲染或粒子,别做物理。
  - 摄像头 → 必须留"跳过直接命名"的降级路径(有人不愿开 / 识别失败)。

## 文案与情绪安全原则(贯穿全程)
1. 机器永远"猜测",用户永远有否定权。
2. 措辞:陪伴、肯定、不评判;不下诊断,不说"你应该"。
3. 节奏慢,每幕都能跳过/回退。
4. 保留裂痕>抹平裂痕:金缮 > 烧成全新。
5. 定位成"一次情绪整合的仪式",不是"心理治疗"——诚实,也更打动人。
